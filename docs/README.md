## Architecture Overview

```
┌─────────────────────────────────────────────────────────┐
│                  Your Spring Boot App                   │
│                                                         │
│   ┌──────────────────┐   ┌──────────────────────────┐  │
│   │  REST Controllers│   │  proxy-api-controller    │  │
│   │  (manual)        │   │  (dynamic proxies)       │  │
│   └────────┬─────────┘   └────────────┬─────────────┘  │
│            │                          │                 │
│            └──────────┬───────────────┘                 │
│                       ▼                                 │
│            ┌──────────────────────┐                     │
│            │  JpaService<T, ID>   │  ◄── common-jpa-service
│            │  (JpaServiceImpl)    │                     │
│            └──────────┬───────────┘                     │
│                       │                                 │
│          ┌────────────┴───────────────┐                 │
│          ▼                            ▼                 │
│  ┌───────────────┐          ┌──────────────────────┐   │
│  │ JpaRepository │          │ QueryJpql<T>         │   │
│  │ (Spring Data) │          │ (generated by        │   │
│  └───────────────┘          │  processor/plugin)   │   │
│                             └──────────────────────┘   │
└─────────────────────────────────────────────────────────┘

Build-time tools
  ├── processor-jpa-service   → generates QueryJpql implementations
  └── jpa-service-plugin-generator → generates Service / Repository / Mapper classes
```

The framework separates concerns into three layers:

| Layer | Responsibility |
|---|---|
| **Code generation** (build time) | Produces type-safe JPQL strings, condition maps, and order maps from annotations on entity services |
| **Query execution** (runtime) | `BaseJpaService` builds the final JPQL by appending WHERE/ORDER BY fragments from the generated maps |
| **Dynamic API** (runtime) | `proxy-api-controller` creates Spring MVC controllers from plain Java interfaces without any implementation class |

## common-jpa-service

### Getting Started

Add the dependency:

```xml
<dependency>
    <groupId>com.github.bld-commons</groupId>
    <artifactId>common-jpa-service</artifactId>
    <version>3.0.15</version>
</dependency>
```

Enable the framework in your Spring Boot application:

```java
@SpringBootApplication
@EnableJpaService
public class MyApplication { }
```

`@EnableJpaService` triggers Spring component scanning for all framework beans
(`ReflectionCommons`, `JdbcTemplateService`, etc.).

### JpaService API

`JpaService<T, ID>` is the central interface. It provides:

#### CRUD

| Method | Description |
|---|---|
| `save(T)` | Persists a new entity |
| `update(T)` | Merges an existing entity and returns the managed instance |
| `saveAndFlush(T)` | Persists and immediately flushes to DB |
| `updateAndFlush(T)` | Merges and immediately flushes to DB |
| `delete(T)` | Deletes a single entity |
| `deleteAll(Collection<T>)` | Deletes all entities in the collection |
| `deleteById(ID)` | Deletes by primary key |
| `deleteAndFlush(T)` | Deletes and immediately flushes to DB |
| `saveAll(Collection<T>)` | Persists all entities in the collection |
| `flush()` | Flushes the persistence context |
| `findById(ID)` | Returns the entity or `null` |
| `findAll()` | Returns all entities |
| `count()` | Returns the total entity count |

#### Dynamic queries

| Method | Description |
|---|---|
| `findByFilter(QueryParameter)` | List query with the generated SELECT |
| `findByFilter(QueryParameter, String sql)` | List query with a custom SELECT |
| `countByFilter(QueryParameter)` | Count query |
| `singleResultByFilter(QueryParameter)` | Single entity or `null` |
| `deleteByFilter(QueryParameter)` | Bulk delete matching entities |
| `mapFindByFilter(QueryParameter)` | Results as `Map<ID, T>` |
| `mapKeyFindByFilter(QueryParameter, Class<J>, String key)` | Results as `Map<J, T>` keyed by a custom field |
| `mapKeyListFindByFilter(QueryParameter, Class<J>, String key)` | Results as `Map<J, List<T>>` |
| `findByFilter(NativeQueryParameter, String sql)` | Native SQL query |
| `countByFilter(NativeQueryParameter, String)` | Native SQL count |
| `singleResultByFilter(NativeQueryParameter, String)` | Native SQL single result |

> **Pagination** is supported on all list queries: set `pageSize` and `pageNumber`
> on the `BaseParameter` or directly on `QueryParameter`.

### Dynamic Query Building

The framework builds JPQL queries dynamically at runtime:

1. The **annotation processor** generates a `*QueryJpqlImpl` class for each entity
   containing static JPQL strings and condition/order maps.
2. `JpaServiceImpl` calls `ReflectionCommons.dataToMap()` to extract non-null fields
   from the `BaseParameter` into a `Map<String, Object>`.
3. `BaseJpaService` appends only the condition fragments whose keys are present in
   the parameter map — so only active filters appear in the WHERE clause.
4. One-to-many JOIN FETCH clauses are injected only when the corresponding filter
   parameter is present, preventing unnecessary joins and N+1 problems.

**Example flow:**

```
ProductFilter { name = "Widget", active = true }
    ↓ ReflectionCommons
Map { "name" → "Widget", "active" → true }
    ↓ condition map lookup
WHERE  AND e.name LIKE :name
  AND  AND e.active = :active
    ↓ parameter binding
SELECT DISTINCT product FROM Product product WHERE AND product.name LIKE :name AND product.active = :active
```

### BaseParameter & Filter Annotations

Extend `BaseParameter` to create a typed filter object:

```java
public class ProductFilter extends BaseParameter {

    // Plain equality: AND product.name = :name
    private String name;

    // LIKE with wildcard on both sides, case-insensitive
    @LikeString(likeType = LikeType.LEFT_RIGHT, upperLowerType = UpperLowerType.LOWER)
    private String description;

    // Date shifted +1 day before binding: AND p.expiresAt < :expiresAt
    @DateFilter(addDay = 1)
    private Date expiresAt;

    // IN clause: AND p.categoryId IN (:categoryIds)
    @ListFilter
    private List<Long> categoryIds;

    // Override the JPQL parameter name
    @FieldMapping("createdBy")
    private String authorName;

    // This field is NEVER added to the WHERE clause
    @IgnoreMapping
    private String internalNote;
}
```

#### Filter annotations reference

| Annotation | Target | Effect |
|---|---|---|
| `@LikeString` | `String` field | Wraps value in `%…%` (configurable), optional case transform |
| `@DateFilter` | `Date`/`Calendar` field | Shifts date by configurable offset before binding |
| `@ListFilter` | `Collection` / array field | Generates `IN (…)` clause |
| `@FieldMapping(name)` | Any field | Uses `name` as the JPQL parameter name instead of field name |
| `@IgnoreMapping` | Any field | Excludes the field from the parameter map entirely |
| `@FilterNullValue` | Any field | Controls whether `null` values are included |
| `@ConditionsZones` | Field (via custom annotation) | Groups conditions into named SQL zones (native queries) |

#### Sorting and pagination

```java
ProductFilter filter = new ProductFilter();
filter.setName("Widget");
filter.addOrderBy("name", OrderType.ASC);   // sort key must be in the generated order map
filter.setPageSize(20);
filter.setPageNumber(0);
```

### QueryParameter

`QueryParameter<T, ID>` wraps the filter and carries additional runtime state:

```java
// From a typed filter object
QueryParameter<Product, Long> qp = new QueryParameter<>(filter);

// Manual parameters (no filter object needed)
QueryParameter<Product, Long> qp = new QueryParameter<>();
qp.addParameter("active", true);
qp.addNullable("deletedAt");   // always append the IS-NULL condition for deletedAt

// Tuple parameters (for multi-column comparisons)
TupleParameter tp = new TupleParameter(...);
qp.addParameter("price", tp);
```

---

### Native SQL Queries

For complex queries that cannot be expressed in JPQL, use `NativeQueryParameter`:

```java
NativeQueryParameter<ReportRow, Long> nqp = new NativeQueryParameter<>(ReportRow.class);
nqp.addZoneParameter("filters", "status", "ACTIVE");

List<ReportRow> rows = myService.findByFilter(nqp,
    "SELECT r.id, r.amount FROM report r ${filters}");
```

Column aliases in the SQL result are mapped to fields of `ReportRow` by name via
reflection. Use `@ResultMapping` or a custom `JpaRowMapper<K>` for non-standard mappings.

## processor-jpa-service

The annotation processor generates `*QueryJpqlImpl` classes from service annotations.

Add it to your project as an annotation processor dependency:

```xml
<dependency>
    <groupId>com.github.bld-commons</groupId>
    <artifactId>processor-jpa-service</artifactId>
    <version>3.0.15</version>
    <scope>provided</scope>
</dependency>
```

### @QueryBuilder

Place on the service (interface or class) to trigger code generation:

```java
@QueryBuilder(
    distinct = true,

    // dot-notation relationship path — always generates:
    //   join fetch product.category category
    // For multi-hop paths, e.g. "product.category.department":
    //   join fetch product.category category
    //   join fetch category.department department
    joins = { "product.category" },

    conditions = {
        @ConditionBuilder(
            field      = "product.name",
            operation  = OperationType.LIKE,
            parameter  = "name",
            upperLower = UpperLowerType.LOWER
        ),
        @ConditionBuilder(
            field     = "product.active",
            operation = OperationType.EQUALS,
            parameter = "active"
        ),
        @ConditionBuilder(
            field     = "product.deletedAt",
            operation = OperationType.EQUALS,
            parameter = "deletedAt",
            nullable  = true           // always appended even when value is null
        )
    },
    jpaOrder = {
        @JpqlOrderBuilder(sortKey = "name",      field = "product.name"),
        @JpqlOrderBuilder(sortKey = "createdAt", field = "product.createdAt")
    }
)
public interface ProductService extends JpaService<Product, Long> { }
```

### @ConditionBuilder

Each `@ConditionBuilder` defines one WHERE condition where the processor derives
the JPQL fragment automatically from a field path and an operation type:

| Attribute | Type | Description |
|---|---|---|
| `field` | `String` | JPQL field path, e.g., `"product.name"` or `"product.address.city"` |
| `operation` | `OperationType` | `EQUALS`, `LIKE`, `IN`, `RANGE`, `NOT_EQUALS`, `IS_NULL`, … |
| `parameter` | `String` | Named parameter key, e.g., `"name"` → `:name` |
| `upperLower` | `UpperLowerType` | `NONE`, `UPPER`, `LOWER` — case transform |
| `nullable` | `boolean` | If `true`, always included even when parameter is `null` |

---

### @CustomConditionBuilder

Used inside `customConditions` (JPQL) or `customNativeConditions` (native SQL) when
the standard operation types of `@ConditionBuilder` are not expressive enough.
You write the raw condition fragment directly.

| Attribute | Type | Description |
|---|---|---|
| `condition` | `String` | The raw JPQL or SQL fragment, including the leading `AND`/`OR` |
| `parameter` | `String` | Named parameter key that activates this condition when non-null |
| `type` | `ConditionType` | `SELECT` (default) → `MAP_CONDITIONS`; `DELETE` → `MAP_DELETE_CONDITIONS` |
| `keys` | `String[]` | Zone names for native queries (see below); defaults to `{"default"}` |

#### customConditions — raw JPQL

The condition is added to `MAP_CONDITIONS` (JPQL) and appended to the WHERE clause
when the named parameter is present. Use the entity alias and JPA field names
(not column names), exactly as you would write them in a JPQL query.

```java
@QueryBuilder(
    customConditions = {
        @CustomConditionBuilder(
            condition = "and genere.idGenere in (:genereId)",
            parameter = "genereId"
            // type defaults to ConditionType.SELECT → goes into MAP_CONDITIONS
            // keys not needed for JPQL conditions
        )
    }
)
```

The processor adds this entry to `MAP_CONDITIONS`:

```java
map.put("genereId", " and genere.idGenere in (:genereId) ");
```

At runtime, the condition is appended to the WHERE clause only when
`genereId` is non-null in the `QueryParameter`.

#### customNativeConditions — raw SQL with zones

The condition is added to one or more zone sub-maps inside `MAP_NATIVE_CONDITIONS`.
At runtime each zone is substituted into the SQL template via the `${zoneName}` placeholder.

```java
@QueryBuilder(
    customNativeConditions = {
        // Tuple comparison — impossible to express with @ConditionBuilder
        @CustomConditionBuilder(
            condition = "and (g.id_genere, pc.id_postazione_cucina) in (:genereTuple)",
            parameter = "genereTuple",
            keys      = {"zone1", "zone2"}   // registered in both zones
        ),
        @CustomConditionBuilder(
            condition = "and g.id_genere in (:idGenere)",
            parameter = "idGenere",
            keys      = {"zone1", "zone2"}
        )
    }
)
```

Generated maps (from the real `GenereQueryJpqlImpl`):

```java
private static Map<String, String> getZone1() {
    Map<String, String> map = new HashMap<>();
    map.put("genereTuple", " and (g.id_genere, pc.id_postazione_cucina) in (:genereTuple) ");
    map.put("idGenere",    " and g.id_genere in (:idGenere) ");
    return map;
}

private static Map<String, String> getZone2() { /* identical */ }

private static Map<String, Map<String, String>> getMapNativeConditions() {
    Map<String, Map<String, String>> map = new HashMap<>();
    map.put("zone1", getZone1());
    map.put("zone2", getZone2());
    return map;
}
```

SQL template usage:

```sql
SELECT g.*, pc.id_postazione_cucina
FROM   genere g
JOIN   postazione_cucina pc ON pc.id_postazione_cucina = g.id_postazione_cucina
${zone1}
```

At runtime, if `genereTuple` is non-null, the zone expands to:

```sql
and (g.id_genere, pc.id_postazione_cucina) in (:genereTuple)
```

### Generated Code

The processor produces a class like (based on the real `GenereQueryJpqlImpl` generated output):

```java
@Component
public class ProductQueryJpqlImpl extends QueryJpql<Product> {

    // FROM clause: the processor walks "product.category" and generates one
    // JOIN FETCH per hop, using the field name as alias
    private static final String FROM_BY_FILTER =
        " From Product product " +
        " join fetch product.category category ";

    private static final String SELECT_BY_FILTER =
        "select distinct product" + FROM_BY_FILTER;

    private static final String COUNT_BY_FILTER =
        "select distinct count(product)" + FROM_BY_FILTER;

    private static final String SELECT_ID_BY_FILTER =
        "select distinct product.id " + FROM_BY_FILTER;

    private static final String DELETE_BY_FILTER =
        "delete from Product product ";

    // Conditions added only when the parameter is non-null (or nullable = true)
    private static Map<String, String> getMapConditions() {
        Map<String, String> map = new HashMap<>();
        map.put("name",   " and upper(product.name) like :name ");
        map.put("active", " and product.active = :active ");
        return map;
    }

    // Conditional LEFT JOIN FETCH — added to the query only when the
    // related filter parameter (e.g. "tagIds") is present
    @Override
    public void mapOneToMany() {
        addJoinOneToMany("tagIds", " left join fetch product.tags tags ");
    }

    @Override public String selectByFilter()              { return SELECT_BY_FILTER; }
    @Override public String countByFilter()               { return COUNT_BY_FILTER; }
    @Override public Map<String, String> mapConditions()  { return MAP_CONDITIONS; }
    // ...
}
```

> **Key distinction:**
> - `joins` → **always-present** `JOIN FETCH` (eager, unconditional) — path traversal generates one clause per hop
> - `mapOneToMany()` → **conditional** `LEFT JOIN FETCH` — added only when the named filter parameter is non-null, preventing unnecessary joins and N+1 issues

## jpa-service-plugin-generator

The Maven plugin generates `Service`, `Repository`, and MapStruct `Mapper` source
files from entity classes.

```xml
<plugin>
    <groupId>com.github.bld-commons</groupId>
    <artifactId>jpa-service-plugin-generator</artifactId>
    <version>3.0.15</version>
    <executions>
        <execution>
            <goals><goal>jpa-service-generator</goal></goals>
        </execution>
    </executions>
    <configuration>
        <persistencePackage>com.example.domain</persistencePackage>
        <repositoryPackage>com.example.repository</repositoryPackage>
        <servicePackage>com.example.service</servicePackage>
        <basePackage>com.example</basePackage>
    </configuration>
</plugin>
```

#### Configuration parameters

| Parameter | Required | Default | Description |
|---|---|---|---|
| `persistencePackage` | yes | — | Package containing `@Entity` classes |
| `servicePackage` | yes | — | Output package for generated services |
| `repositoryPackage` | no | same as service | Output package for generated repositories |
| `basePackage` | yes | — | Root package for imports |
| `outputDirectory` | no | `src_main_java` | Where generated sources are written |

> **Note:** Consider pointing `outputDirectory` to `target/generated-sources/java` to
> avoid committing generated code to version control. Add that directory to the
> compile source roots in your `pom.xml`.

## proxy-api-controller

Generates Spring MVC REST controllers at runtime from annotated interfaces —
no implementation class required.

```xml
<dependency>
    <groupId>com.github.bld-commons</groupId>
    <artifactId>proxy-api-controller</artifactId>
    <version>3.0.15</version>
</dependency>
```

### Setup

```java
@SpringBootApplication
@EnableProxyApiController
public class MyApplication { }
```

### Annotations

#### @ApiFindController

Marks an interface as a dynamic REST controller. The framework creates a Java
dynamic proxy that intercepts all method calls.

```java
@ApiFindController
@RequestMapping("/api/products")
@ApiFind(entity = Product.class, id = Long.class)
@ApiMapper(value = ProductMapper.class, method = "toDto")
public interface ProductController {

    @PostMapping("/search")
    List<ProductDto> search(@RequestBody ProductFilter filter);

    @PostMapping("/search/page")
    List<ProductDto> searchPaged(@RequestBody ProductFilter filter,
                                  @RequestParam int page,
                                  @RequestParam int size);
}
```

#### @ApiFind

Binds the controller (or a single method) to a JPA entity and its primary-key type.
Can be placed at class or method level; method-level takes precedence.

```java
@ApiFind(entity = Product.class, id = Long.class)
```

#### @ApiMapper

Specifies the mapper used to convert query results before the response is returned.

```java
@ApiMapper(value = ProductMapper.class, method = "toDto")
```

If `method` is blank, the framework resolves the method by matching the entity type
to the mapper method's parameter type.

#### @ApiBeforeFind

Registers a hook invoked **before** the query executes.

```java
@ApiBeforeFind(TenantContextInjector.class)
```

```java
@Component
public class TenantContextInjector implements BeforeFind<ProductFilter, Long> {
    @Override
    public void before(QueryParameter<Product, Long> qp, ProductFilter filter) {
        qp.addParameter("tenantId", SecurityContext.getTenantId());
    }
}
```

#### @ApiAfterFind

Registers a hook invoked **after** the query returns results.

```java
@ApiAfterFind(ProductEnricher.class)
```

```java
@Component
public class ProductEnricher implements AfterFind<ProductDto> {
    @Override
    public void after(List<ProductDto> results) {
        results.forEach(dto -> dto.setDisplayLabel(dto.getName() + " — " + dto.getSku()));
    }
}
```

### Hooks

| Interface | When | Signature |
|---|---|---|
| `BeforeFind<F, ID>` | Before query | `void before(QueryParameter<T,ID> qp, F filter)` |
| `AfterFind<R>` | After query, before response | `void after(List<R> results)` |

Both hook classes must be Spring beans (`@Component` or `@Service`) so the
framework can retrieve them from the application context.

## Full Example

### 1. Entity

```java
@Entity
@Table(name = "product")
public class Product {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
    private Boolean active;

    @ManyToOne(fetch = FetchType.LAZY)
    private Category category;
    // ...
}
```

### 2. Service interface (with @QueryBuilder)

```java
@QueryBuilder(
    distinct = true,
    // dot-notation path → always generates: join fetch product.category category
    joins = { "product.category" },
    conditions = {
        @ConditionBuilder(field = "product.name",   operation = OperationType.LIKE,   parameter = "name"),
        @ConditionBuilder(field = "product.active", operation = OperationType.EQUALS, parameter = "active")
    },
    jpaOrder = {
        @JpqlOrderBuilder(sortKey = "name", field = "product.name")
    }
)
public interface ProductService extends JpaService<Product, Long> { }
```

### 3. Service implementation (generated or manual)

```java
@Service
public class ProductServiceImpl extends JpaServiceImpl<Product, Long>
        implements ProductService {

    @Autowired private ProductRepository repository;
    @PersistenceContext private EntityManager em;

    @Override protected JpaRepository<Product, Long> getJpaRepository() { return repository; }
    @Override protected EntityManager getEntityManager() { return em; }
}
```

### 4. Filter object

```java
public class ProductFilter extends BaseParameter {

    @LikeString(likeType = LikeType.LEFT_RIGHT, upperLowerType = UpperLowerType.LOWER)
    private String name;

    private Boolean active;

    // getters / setters
}
```

### 5. Dynamic REST controller (proxy)

```java
@ApiFindController
@RequestMapping("/api/products")
@ApiFind(entity = Product.class, id = Long.class)
@ApiMapper(value = ProductMapper.class, method = "toDto")
public interface ProductController {

    @PostMapping("/search")
    List<ProductDto> search(@RequestBody ProductFilter filter);
}
```

### 6. Using the service manually

```java
@RestController
@RequestMapping("/api/products/manual")
public class ManualProductController {

    @Autowired private ProductService productService;

    @PostMapping("/search")
    public List<Product> search(@RequestBody ProductFilter filter) {
        filter.addOrderBy("name", OrderType.ASC);
        filter.setPageSize(50);
        filter.setPageNumber(0);
        return productService.findByFilter(new QueryParameter<>(filter));
    }

    @GetMapping("/{id}")
    public Product get(@PathVariable Long id) {
        return productService.findById(id);
    }
}
```
